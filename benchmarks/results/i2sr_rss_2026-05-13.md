# I2_SR RSS Probe, 2026-05-13

Hardware: Intel Xeon Silver 4116, portable AVX2 llama.cpp build, no BLAS.
The runtime was built with `patches/llama-i2sr-row-scale-qtype.patch` applied
for this measurement, then the patch was reversed and the default source tree
was rebuilt.

Command:

```bash
python benchmarks/run_gguf_memory_probe.py \
  --models-json benchmarks/gguf_qwen15b_i2sr_x86act_manifest.json \
  --out-dir benchmark_results/gguf-rss-qwen15b-i2sr-x86act-context-2026-05-13 \
  --llama-bin-dir build-portable-avx2/bin \
  --threads 12 \
  --ctx-sizes 512 2048 8192 32768 \
  --tokens 1
```

The probe runs `llama-cli` under `/usr/bin/time -v`, with `-c 512`, 12 threads,
CPU-only execution, and one generated token so the model is loaded and used.
The reported metric is peak resident set size from `/usr/bin/time`.

| artifact | file MiB | max RSS GiB | return code |
| --- | ---: | ---: | ---: |
| fixed row-scale `I2_SR` x86 ACT candidate, `ctx=512` | 1,211.3 | 1.250 | 0 |
| fixed row-scale `I2_SR` x86 ACT candidate, `ctx=2048` | 1,211.3 | 1.291 | 0 |
| fixed row-scale `I2_SR` x86 ACT candidate, `ctx=8192` | 1,211.3 | 1.455 | 0 |
| fixed row-scale `I2_SR` x86 ACT candidate, `ctx=32768` | 1,211.3 | 2.114 | 0 |

## Interpretation

The fixed direct `I2_SR` artifact has the same measured RSS class as the
row-scale `I2_S` prototype across this context sweep. At `ctx=32768`, fixed
`I2_SR` reports `2.114 GiB`, matching the row-scale `I2_S` prototype and close
to row-scale `TQ2_0` (`2.121 GiB`). This supports the memory-side claim that
the separate-qtype candidate preserves the row-scale ternary memory advantage
over FP16 while remaining larger than the dense `Q4_K_M` baseline.
